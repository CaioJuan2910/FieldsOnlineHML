🐛 Bugs Encontrados e Corrigidos

Bug #001
VSync Invertido
🔴 Crítico
📄 Arquivo: [VS] Settings_File.rb
🏷️ Severidade: Crítico
⚠️ Impacto: VSync ficava DESATIVADO quando o jogador ativava, e ATIVADO quando desativava. O comportamento era exatamente o oposto do esperado em 100% dos casos.
❌ Código Original (ERRADO)
Graphics.vsync = (vsync == 0) # Ativa VSync quando valor é 0 (desligado) — invertido!
✅ Código Modernizado (CORRETO)
Graphics.vsync = (@vsync == 1) # 1 = ativado, 0 = desativado — semântica correta
💡 Explicação: O código original usava == 0 para ativar o VSync, o que é semanticamente invertido. O valor 0 representa "desligado" e 1 representa "ligado". A correção usa == 1, tornando a lógica direta e intuitiva.
Bug #002
Fullscreen com Lógica Completamente Invertida
🔴 Crítico
📄 Arquivo: [VS] Settings_File.rb
🏷️ Severidade: Crítico
⚠️ Impacto: O toggle era acionado quando já estava no estado correto, e NÃO era acionado quando precisava mudar. Em todos os 4 casos possíveis, o comportamento era o oposto do esperado.
❌ Código Original (ERRADO)
Graphics.toggle_fullscreen if fullscreen == 0 && !Graphics.is_fullscreen? || fullscreen == 1 && Graphics.is_fullscreen?
🔍 O que acontecia em cada caso:
❌
Quer janela (0) + já em janela → TOGLA (errado — já está correto, não deveria mudar!)
❌
Quer janela (0) + em fullscreen → NÃO togla (errado — deveria mudar para janela!)
❌
Quer fullscreen (1) + em janela → NÃO togla (errado — deveria mudar para fullscreen!)
❌
Quer fullscreen (1) + já em fullscreen → TOGLA (errado — já está correto, não deveria mudar!)
✅ Código Modernizado (CORRETO)
currently_fullscreen = Graphics.is_fullscreen? should_be_fullscreen = (@fullscreen == 1) Graphics.toggle_fullscreen if currently_fullscreen != should_be_fullscreen
💡 O que faz agora: Compara o estado ATUAL com o estado DESEJADO. Só aciona o toggle se forem diferentes. Lógica clara, legível e correta em todos os 4 casos possíveis.

⚠️ Observação de Design: FORBIDDEN_NAMES

⚠️ Comportamento Intencional por Design

O comportamento de não bloquear "Admin123" (sem espaço) é intencional por design. O espaço trailing em cada termo ('adm ', 'gm ') foi adicionado deliberadamente para evitar falsos positivos em nomes legítimos.

Este é um padrão de filtragem por prefixo com delimitador, onde o espaço funciona como separador de palavra, garantindo que apenas nomes que começam com o termo seguido de espaço sejam bloqueados.

🚫
"adm João" → BLOQUEADO — contém 'adm ' (com espaço)
✅
"Admirable" → PERMITIDO — não contém 'adm ' (sem espaço após)
🚫
"gm Caio" → BLOQUEADO — contém 'gm ' (com espaço)
✅
"Admin123" → PERMITIDO — não contém 'adm ' (sem espaço)
📌 Recomendação: Documentar este comportamento explicitamente no código para evitar confusão futura — já implementado na versão modernizada com comentário detalhado na seção FORBIDDEN_NAMES.
