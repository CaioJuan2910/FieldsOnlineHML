🚀 Passo a passo de instalação
⌨️
1

Abrir o Editor de Scripts

Com o projeto aberto no RPG Maker VX Ace, pressione F11 para abrir o Editor de Scripts.

Você verá a lista de scripts no painel esquerdo.

🖱️
2

Localizar [VS] Game_Player

Role a lista até encontrar [VS] Game_Player.

Clique com o botão direito sobre ele e selecione Inserir para criar um novo script logo abaixo.

✏️
3

Nomear o novo script

No campo de nome (topo do painel direito), apague o texto padrão e digite exatamente:

[FO] Smooth_Movement

O nome é importante para identificação e organização.

📋
4

Colar o código e salvar

Cole o conteúdo completo do script no editor de texto à direita.

Clique em OK para confirmar. O script estará ativo na próxima vez que o jogo for iniciado.

⚙️ Configurações rápidas

As constantes abaixo ficam no topo do script, na seção CONFIG. Ajuste conforme a necessidade do seu servidor:

⚙️ Constante 📌 Valor padrão 📝 O que faz
NET_LERP_SPEED 0.20 Suavidade de outros jogadores online. Valores menores = mais suave, porém com mais atraso.
EVENT_LERP_SPEED 0.30 Suavidade de NPCs e eventos com rota de patrulha.
MOB_LERP_SPEED 0.25 Suavidade de MOBs e inimigos ao se aproximar do jogador.
SNAP_THRESHOLD 0.01 Distância mínima para "snap" final — evita que o personagem fique oscilando no destino.
SMOOTH_LOCAL_PLAYER false Se true, aplica suavização também ao jogador local (não recomendado para a maioria dos casos).
💡 Dica: Valores de LERP entre 0.15 e 0.30 costumam oferecer o melhor equilíbrio entre fluidez e responsividade em servidores com latência normal.
✅ O que testar após instalar
Progresso dos testes:
0 / 5 concluídos
🏃 Mover o personagem pelo mapa — o movimento deve parecer mais fluido e sem "saltos" entre tiles.
🤖 Observar NPCs com rota de patrulha — devem se mover suavemente entre os pontos da rota.
👾 Observar MOBs se aproximando — inimigos não devem mais "saltar" entre tiles ao perseguir o jogador.
🌐 Conectar dois clientes — abra dois clientes e observe o outro jogador se mover suavemente na tela.
🔍 Verificar o console de erros — pressione F8 durante o jogo e confirme que não há erros relacionados ao script.
🛠️ Solução de problemas
⚠️

Problemas comuns e como resolver

🔴
NoMethodError: undefined method 'screen_x'
Causa: O script está na posição errada na lista.
Solução: Mova [FO] Smooth_Movement para ficar imediatamente após [VS] Game_Player e antes de qualquer script de Sprite.
🟡
Personagens ficando "escorregando" ou atrasados
Causa: Valor de LERP muito baixo para a latência do servidor.
Solução: Aumente NET_LERP_SPEED para 0.35 ~ 0.50 no topo do script e reinicie o jogo.
🟠
Movimento parece igual ao anterior (sem suavização)
Causa: O script pode não ter sido salvo corretamente ou o jogo não foi reiniciado.
Solução: Abra o Editor de Scripts (F11), confirme que o código está presente, clique em OK e reinicie o jogo completamente.
