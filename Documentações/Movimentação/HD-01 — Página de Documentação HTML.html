2 ✅ Código Corrigido — DEPOIS 
def handle_player_movement(client, buffer)
  d = buffer.read_byte
  # Anti-speed hack
  #return unless client.movable?
  return if d < Enums::Dir::DOWN_LEFT || d > Enums::Dir::UP_RIGHT
  #return if client.has_text?

  # [HD-01] stop_count diferenciado: diagonal (d ímpar) espera 0.240s
  # enquanto movimento cardinal espera 0.170s, normalizando a velocidade efetiva.
  # d.odd? detecta diagonais: 1, 3, 7, 9 são ímpares (diagonais) | 2, 4, 6, 8 são pares (cardinais)
  client.stop_count = Time.now + (d.odd? ? 0.240 : 0.170)   # ← LINHA CORRIGIDA

  #if d.odd?
  #client.move_diagonal(d)
  #else
  client.move_straight(d)
  #end
  if client.move_succeed
    client.check_floor_effect
    client.check_touch_event
    client.close_windows
  end
end

3 🧮 Lógica da Correção
🎮 Mapa Numpad — RPG Maker

8↑ par
9↗ ímpar
4← par
5—
6→ par
1↙ ímpar
2↓ par

Diagonal (ímpar) — 0.240s
Cardinal (par) — 0.170s
📐 Cálculo Matemático
0.170s × √2 ≈ 0.240s
0.170 × 1.41421 = 0.24041 → 0.240s
A diagonal percorre √2 ≈ 1.414 tiles por passo.
Para manter a mesma velocidade efetiva, o delay deve ser proporcional à distância percorrida.
velocidade = distância / tempo
5.88 = 1.000 / 0.170
5.89 = 1.414 / 0.240

5 📌 Observação Importante
💡 O bloco diagonal está comentado no código original do VXA-OS. A correção HD-01 é aplicada preventivamente para quando o módulo [FO] Diagonal_Movement for ativado no projeto Fields Online. Mesmo com o bloco comentado, a linha de stop_count já é executada para todos os movimentos — portanto a correção é válida e necessária desde já, garantindo que nenhum jogador explore a diferença de velocidade mesmo no estado atual do servidor.
